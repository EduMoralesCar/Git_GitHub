
## Buenas prácticas con Git y GitHub

- Haz commits pequeños y descriptivos.
- Usa ramas para cada nueva funcionalidad o corrección.
- Mantén tu rama actualizada antes de hacer merge.
- Resuelve conflictos de inmediato y revisa los cambios.
- No subas archivos sensibles o innecesarios; usa `.gitignore`.
- Agrega README.md y LICENSE a tus proyectos.
- Utiliza tags para marcar versiones importantes.
- Protege la rama principal con revisiones y reglas.
- Activa la verificación en dos pasos en GitHub.
- Comunica los cambios importantes con tu equipo.
